
# 개발
- 서버
	- ### iOS 서버 작업 — AASA 파일 호스팅 〔담당: 사용자/서버〕
		`apple-app-site-association`(AASA) 파일을 서버에 올려야 합니다. 이게 "이 도메인의 어떤 경로를 앱이 가로챌지" Apple에 알려주는 핵심입니다.
		
		- **호스팅 위치**: `https://barnote.net/.well-known/apple-app-site-association`
		- **파일 내용**:
		
		```
		{  "applinks": {    "details": [      {        "appIDs": ["Z97K7ZHNKX.com.oq.barnote"],        "components": [          { "/": "/note/*", "comment": "노트 상세" },          { "/": "/user/*", "comment": "유저 노트 목록" }        ]      }    ]  }}
		```
		
		- **필수 조건** (하나라도 어기면 동작 안 함):
		    - 확장자 **없이** 제공 (`.json` 붙이면 안 됨)
		    - 응답 헤더 `Content-Type: application/json`
		    - **HTTPS**로 **200 OK** 직접 응답 (리다이렉트 ❌, 인증/쿠키 ❌)
		    - `barnote.net`으로 접속 시 응답해야 함 (www를 쓴다면 www에서도 응답 필요)
	- ## 📋 AOS 서버작업
		**1. 릴리즈 서명 SHA-256 지문 확보**
		- Play App Signing 사용 중이면 → **Play Console → 앱 → 설정 → 앱 무결성 → "앱 서명 키 인증서"** 의 SHA-256 복사. (+ "업로드 키 인증서" SHA-256 도 같이 넣으면 내부테스트 빌드에서도 검증됨)
		- keystore 직접: `keytool -list -v -keystore <release.keystore> -alias <alias>` → `SHA256:` 줄
		- (로컬 디버그 빌드로 테스트하려면 `~/.android/debug.keystore` 지문도 임시로 추가)
		
		**2. `assetlinks.json` 작성**
		- 템플릿의 `REPLACE_WITH_...` 자리를 1번 지문으로 치환. `package_name`은 `com.oq.barnote` 로 이미 맞음. 지문은 배열로 여러 개(릴리즈/업로드/디버그) 넣어도 됨.
		
		**3. 서버에 호스팅** ← 가장 중요한 누락 지점
		- 정확히 `https://barnote.net/.well-known/assetlinks.json` 경로
		- **HTTPS**, `Content-Type: application/json`, **리다이렉트 없이 200 OK** 로 직접 응답 (App Links는 리다이렉트 거부), 인증·robots 차단 없이 공개 접근
		
		**4. `http` → `https` 리다이렉트 (웹서버)**
		- 주신 링크가 `http://` 인데, App Links 자동검증은 **https에서만** 동작합니다. 웹서버에서 `http://barnote.net/*` → `https://` **301 리다이렉트**를 걸면 http 링크도 결국 앱으로 열립니다. (안 걸면 http 링크는 그냥 브라우저로 열림)
	- 공개 안함에서 전체 공개로 수정했을때 해당 제품상세에서 다른 유저에게 노트가 노출이 안된다

- App 모든 플랫폼
	- iOS 검증 〔담당: 사용자〕
		- AASA 배포 후 `https://barnote.net/.well-known/apple-app-site-association`를 브라우저나 [Apple AASA Validator](https://search.developer.apple.com/appsearch-validation-tool/)로 확인.
		- **실기기**에서 테스트 (시뮬레이터는 universal link 신뢰성이 낮음). 메모 앱 등에 링크를 붙여넣고 탭하면 앱이 열려야 합니다. (Safari 주소창에 직접 입력하면 앱으로 안 가니, **링크를 탭**해서 테스트하세요.)
	- **AOS 검증 & 테스트**
		- assetlinks 확인: `https://digitalassetlinks.googleapis.com/v1/statements:list?source.web.site=https://barnote.net&relation=delegate_permission/common.handle_all_urls`
		- 검증 상태: `adb shell pm get-app-links com.oq.barnote` → `barnote.net: verified` 떠야 정상 / 재검증: `adb shell pm verify-app-links --re-verify com.oq.barnote`
		- 실동작: `adb shell am start -a android.intent.action.VIEW -d "https://barnote.net/note/<UUID>"` → 앱 열림 확인, 미설치 시 브라우저 폴백 확인
